Daten importieren: Produkte, Kunden und Bestand (CSV)

Admin

Produktkatalog, Kundendatei UND Bestand aus einer Datei importieren oder aktualisieren, ohne Neueingabe und ohne Duplikate.

Präsentation

Der Import fügt viele Daten auf einmal hinzu oder aktualisiert sie aus einer CSV-Datei (eine Tabelle aus Excel, Google Sheets oder LibreOffice). Der Bildschirm hat drei Registerkarten: Produkte, Kunden und Bestand (letztere mit zwei Unterkarten: Bestandsmengen und Bewegungen). Ideal für Onboarding oder Massenkorrektur. Der Server prüft jede Zeile: gültige Zeilen werden importiert, fehlerhafte werden mit klarer Erklärung aufgelistet - nichts geht still verloren.

Zugang

Administrator-Dashboard, Bereich Bestand, Karte "Daten importieren", dann Registerkarte Produkte oder Kunden wählen. (Nur Administratoren.)

Beschreibung der Oberfläche

  • Registerkarten "Produkte", "Kunden" und "Bestand": Oben drei Registerkarten. Produkte importiert Ihren Katalog; Kunden importiert Ihre Kundendatei; Bestand aktualisiert die Mengen VORHANDENER Produkte, mit zwei Unterkarten: "Bestandsmengen" (Sie geben den neuen Gesamtbestand nach einer Zählung an) und "Bewegungen" (Sie erfassen Zu-/Abgänge). Jede Registerkarte funktioniert gleich (Vorlage herunterladen, ausfüllen, importieren) und behält ihren Bericht beim Wechseln.
  • Bestandsspalten (Registerkarte Bestand): Das Produkt wird per 'Referenz' (oder 'Name') identifiziert: es muss BEREITS existieren (der Bestandsimport legt nie ein Produkt an; nutzen Sie die Registerkarte Produkte). Unterkarte Bestandsmengen: 'Menge' = der neue Gesamtbestand (eine protokollierte Inventurbewegung wird für die Differenz erstellt). Unterkarte Bewegungen: 'Bewegungstyp' = Eingang oder Ausgang, 'Menge' = die Bewegungsmenge; ein Ausgang größer als der verfügbare Bestand wird abgelehnt (nie negativer Bestand). 'Grund' ist optional.
  • Kundenspalten (Registerkarte Kunden): Auf der Registerkarte Kunden: 'Vorname', 'Nachname' und 'Telefon' sind Pflicht (Stern). Die Telefonnummer muss im internationalen Format sein (z. B. +33612345678); sie erkennt auch einen vorhandenen Kunden und aktualisiert ihn (kein Duplikat). 'E-Mail' und 'Adresse' sind optional. Der Treuepunktestand wird nie importiert (er wird automatisch durch Verkäufe verwaltet).
  • Schaltfläche "Vorlage herunterladen": Lädt eine fertige CSV-Datei mit den richtigen Spalten (in Ihrer Sprache) und zwei Beispielen herunter. Empfohlener Startpunkt: ersetzen Sie einfach die Beispiele durch Ihre Produkte.
  • Erwartete Spalten: Die Liste der Dateispalten. Nur 'Produktname' und 'Verkaufspreis' sind Pflicht (Stern). 'Referenz (SKU)' erkennt ein vorhandenes Produkt und aktualisiert es. Andere Spalten (Einkaufspreis, Bestand, Schwellen) sind optional.
  • Bericht (Erstellt / Aktualisiert / Ignoriert): Drei Zähler fassen den Import zusammen. Darunter wird jede fehlerhafte Zeile mit Nummer und Grund aufgelistet. Korrigieren und erneut starten: keine Duplikate.

Schritt-für-Schritt-Anleitung

  1. Aktion: Wählen Sie oben die Registerkarte: "Produkte" für den Katalog, "Kunden" für die Kundendatei. Ergebnis: Die gewählte Registerkarte zeigt ihre erwarteten Spalten und Schaltflächen.
  2. Aktion: Tippen Sie auf "Vorlage herunterladen": eine fertige CSV-Datei (mit den richtigen Spalten in Ihrer Sprache + 2 Beispielen) wird auf Ihr Gerät heruntergeladen. Ergebnis: Sie erhalten eine Vorlagendatei, bereits im richtigen Format.
  3. Aktion: Öffnen Sie die Vorlage in Excel (oder Google Sheets), ersetzen Sie die 2 Beispielzeilen durch Ihre Produkte und speichern Sie die Datei. Ergebnis: Ihre Datei ist bereit zum Import. Prüfung: Lassen Sie die erste Zeile (die Spaltentitel) unverändert; füllen Sie mindestens Name und Verkaufspreis aus.
  4. Aktion: Öffnen Sie "Produkte importieren", tippen Sie auf "CSV-Datei wählen" und wählen Sie Ihre Datei. Ergebnis: Der Import läuft und der Bericht erscheint. Prüfung: Prüfen Sie die Zähler Erstellt / Aktualisiert / Ignoriert.
  5. Aktion: Wenn Zeilen ignoriert wurden, korrigieren Sie sie (der Grund wird angezeigt) und starten Sie den Import erneut. Ergebnis: Bereits importierte Produkte werden aktualisiert, nicht dupliziert; korrigierte Zeilen werden hinzugefügt.

Beispiele

[Simple] Erster Katalogimport: Eine Datei mit 150 Produkten (Referenz, Name, Verkaufspreis, Menge). Ergebnis: 150 erstellt, 0 ignoriert. Der Anfangsbestand wird berücksichtigt.

Sonderfälle

Der erneute Import DERSELBEN Datei erzeugt keine Duplikate: erkannte Produkte (per Referenz, sonst per Name) werden aktualisiert. Beliebig oft korrigieren und erneut starten.

Ein Dezimalkomma wird akzeptiert (12,50 wie 12.50). Die Datei darf höchstens 2 MB und 2000 Zeilen umfassen; sonst aufteilen.

Fehlermeldungen

Der Produktname ist erforderlich.
Ursache: Die Spalte Name ist in dieser Zeile leer. Lösung: Geben Sie einen Namen ein und starten Sie erneut.
Ungültige Zahl.
Ursache: Preis oder Menge enthält nicht-numerische Zeichen. Lösung: Verwenden Sie eine Zahl (z. B. 12,50) ohne Währungssymbol.
Pflichtspalten fehlen.
Ursache: Die Kopfzeile enthält nicht 'Name' und/oder 'Verkaufspreis'. Lösung: Beginnen Sie erneut mit dem Export Ihrer Produkte als Vorlage.

Tipps

  • Füllen Sie die Spalte Referenz (SKU) aus: damit wird ein vorhandenes Produkt beim erneuten Import zuverlässig gefunden und aktualisiert.
