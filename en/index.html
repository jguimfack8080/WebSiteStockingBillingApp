The scanner, made clear

The barcode scanner: a helpful tool, never a requirement

Scanning speeds up your day, but the app remains fully usable without any hardware. Here are the compatible options, from the simplest to the most complete.

USB scanner (checkout gun)

Just plug it in via USB. In keyboard mode (the standard mode for these devices) it works straight away: the scanned code is entered as if you had typed it.

Wireless Bluetooth or Wi-Fi scanner

Compatible as long as it is paired in keyboard mode on your device. Once connected, it behaves like a USB gun, with no extra setup inside the app.

Network scanner or integrated terminal

Compatible if it sends the code as keyboard input into the active field (keyboard mode or a bridge to the PC). That is the simple condition to check with your hardware.

Your smartphone or tablet camera

On Android and on the Web, the camera acts as a scanner: point at the item and the code is read automatically. Ideal if you have no scanner gun.

Manual code entry

Available everywhere: type the code by hand when that is more convenient. An alternative that is always ready, on every device.

The rule is simple: any scanner that sends readable data (barcodes or QR codes) into the active field of the app is compatible. In practice, the device must deliver the code as keyboard input, which is the standard mode for scanner guns.

RBAC · Access control

A role for every team member

Role-based access control (RBAC): administrator, manager, cashier. And beyond roles, the administrator grants or revokes specific rights, user by user. Permissions are verified server-side on every action - the backend remains the authority.

Administrator

Full control
  • Manages users and their roles
  • Configures the company, VAT and invoicing
  • Full access to reports and the audit log

Manager

Operational steering
  • Tracks stock, thresholds and alerts
  • Views sales, invoices and reports
  • Receives critical-stock notifications

Cashier

Day-to-day sales
  • Takes payment quickly
  • Prints receipts and views their sales
  • Views the product catalogue

Tailored rights, beyond roles

Need a specific manager to edit products, or a trusted cashier to issue refunds? The administrator adjusts rights person by person via a clear matrix (by module and by action). The change affects only that person, and everything stays verified server-side.

Security and trust

Your data protected, by design

Security is not an option. It is built into every layer of the app.

Multi-tenant isolation

Each company's data is strictly partitioned: the company identifier comes from the authentication token, never from the request (anti-IDOR protection). One company never sees another's.

RBAC + token authentication

Authentication via a secure token and role-based access control (RBAC): every sensitive action is authorised server-side, never on client trust alone.

Tamper-proof audit trail

Sensitive actions (sign-ins, edits, stock movements) are traced, timestamped and chained by hash : any tampering with the log is detected. Evidential traceability.

Encrypted passwords

Passwords are hashed. Resets use a one-time code sent by email.

Server-side validation

All business rules and calculations (stock, VAT, totals) are validated by the server, never by the client alone.

Input protection

Parameterised queries against injection, strict validation of all input and escaping of output. Security is checked at every layer.

BizCoRa is not just a checkout app. It is a multi-tenant SaaS built on engineering foundations: microservices, role-based access control (RBAC), exact monetary calculations and data pre-aggregation. RBAC (role-based access control)

Three roles (administrator, manager, cashier), each with a precise scope. Permissions are verified server-side on every sensitive action (authorisation dependencies), never on client trust alone.

Strict tenant isolation

Each company (tenant) is fully partitioned. The company identifier is injected from the authentication token, never from the request: no data can leak from one tenant to another (protection against indirect access / IDOR).

Decoupled microservices

Authentication, inventory and sales are independent services, each with its own database, deployed and isolated separately. Zero circular dependencies, scaling service by service.

Designed for millions of rows

Pre-aggregated statistics (daily summaries), keyset pagination (no deep OFFSET), composite indexes and targeted FORCE INDEX . The result: instant reports, from day one to decades of activity.

Exact monetary calculations

All amounts are in DECIMAL (never floating point): net revenue, COGS, gross margin, VAT collected / deductible / due, weighted average cost (WAC). The result is exact, to the cent, on every calculation.

Tamper-proof audit trail

The log of sensitive operations is immutable and hash-chained : any tampering is detected. Evidential traceability, timestamped in UTC and rendered in the company's timezone.
