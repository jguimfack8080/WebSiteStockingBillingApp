Create an employee and invite them to join

Admin

Two ways to create an employee account: email invitation or temporary password.

Presentation

When you create an employee, BizCoRa automatically chooses the onboarding method based on whether you provided an email address.

Option A - Email provided: an activation email is sent automatically. The employee clicks the link in the email, sets their own password and their account is activated.

Option B - No email: the account is created active with a temporary password. This password is shown ONLY ONCE: note it and share it privately with the employee. On first login, they will be required to set a new personal password.

Access

Administrator dashboard > Administration > Manage users > "New user" button (+ icon at top right of the list).

Interface description

  • First name field: Employee first name (required).
  • Last name field: Employee last name (required).
  • Email field (optional): If you fill in the email, BizCoRa automatically sends an activation link. If you leave this field empty, a temporary password is generated.
  • ID card number: Employee legal identifier (required).
  • Birth date: Select the date via the calendar (required).
  • Role: Cashier, Manager or Administrator. Defines the default permissions.

Step-by-step procedure

  1. Action: From the user list, tap the "+" button at the top right. Result: The creation form opens.
  2. Action: Enter the first name, last name, ID card number, birth date and role. The email is optional: leave it blank if the employee has no professional email address. Result: The form is complete and ready to submit.
  3. Action: Tap "Create". Result: BizCoRa creates the account and shows the result: email sent (Option A) or temporary password (Option B).
  4. Action: Option A: read the email address shown in the confirmation window. Result: The employee receives an email with an activation link valid for 72 hours. After that, resend the invitation from their profile.
  5. Action: Option B: in the result window, tap the copy button next to the temporary password. Result: The password is copied to the clipboard. Share it with the employee privately (call, encrypted message, in person). It will not be visible after closing.
  6. Action: The employee logs in with their credentials (email link for Option A, temporary password for Option B). Result: Option A: the activation screen opens, the employee sets their password and the account becomes active. Option B: the employee is automatically redirected to set their personal password before accessing the app.

Error messages

"This activation link is invalid or has expired".
Cause: The link has been used once (it is single-use) or 72 hours have elapsed. Solution: Go to the employee's profile and use "Resend invitation" to generate a new link.

Tips

  • The activation link expires after 72 hours. If the employee has not used it, go to their profile and tap "Resend invitation".
  • The temporary password is only visible once. It cannot be retrieved afterwards. If the employee lost it, deactivate the account and create a new one.
  • An employee without an email can still receive an invitation if you update their email later in their profile.
