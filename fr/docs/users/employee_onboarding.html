Créer un employé et l'inviter à rejoindre

Admin

Deux manières de créer un compte employé : invitation par email ou mot de passe temporaire.

Présentation

Quand vous créez un employé, BizCoRa choisit automatiquement la méthode d'onboarding selon que vous avez fourni une adresse email ou non.

Option A - Email fourni : un email d'activation est envoyé automatiquement. L'employé clique le lien dans l'email, choisit son propre mot de passe et son compte est activé.

Option B - Pas d'email : le compte est créé actif avec un mot de passe temporaire. Ce mot de passe s'affiche UNE SEULE FOIS : notez-le et communiquez-le en privé à l'employé. À sa première connexion, il devra choisir un nouveau mot de passe personnel.

Accès

Tableau de bord administrateur > Administration > Gérer les utilisateurs > bouton "Nouvel utilisateur" (icône + en haut à droite de la liste).

Description de l'interface

  • Champ Prénom : prénom de l'employé (obligatoire).
  • Champ Nom : nom de famille de l'employé (obligatoire).
  • Champ Email (optionnel) : si vous renseignez l'email, BizCoRa envoie automatiquement un lien d'activation. Si vous laissez ce champ vide, un mot de passe temporaire est généré.
  • Numéro de carte d'identité : identifiant légal de l'employé (obligatoire).
  • Date de naissance : sélectionnez la date via le calendrier (obligatoire).
  • Rôle : Caissier, Manager ou Administrateur. Définit les droits par défaut.

Procédure pas à pas

  1. Action : depuis la liste des utilisateurs, appuyez sur le bouton "+" en haut à droite. Résultat : le formulaire de création s'ouvre.
  2. Action : saisissez le prénom, le nom, le numéro de carte d'identité, la date de naissance et le rôle. L'email est optionnel : laissez-le vide si l'employé n'a pas d'adresse email professionnelle. Résultat : le formulaire est complet et prêt à être envoyé.
  3. Action : appuyez sur "Créer". Résultat : BizCoRa crée le compte et affiche le résultat : email envoyé (Option A) ou mot de passe temporaire (Option B).
  4. Action : Option A : lisez l'adresse email affichée dans la fenêtre de confirmation. Résultat : l'employé reçoit un email avec un lien d'activation valide 72 heures. Passé ce délai, renvoyez l'invitation depuis sa fiche.
  5. Action : Option B : dans la fenêtre de résultat, appuyez sur le bouton de copie à côté du mot de passe temporaire. Résultat : le mot de passe est copié dans le presse-papiers. Communiquez-le à l'employé de façon privée (appel, message chiffré, remise en main propre). Il ne sera plus visible après fermeture.
  6. Action : l'employé se connecte avec ses identifiants (lien email pour l'Option A, mot de passe temporaire pour l'Option B). Résultat : Option A : l'écran d'activation s'ouvre, l'employé définit son mot de passe et le compte devient actif. Option B : l'employé est redirigé automatiquement pour définir son mot de passe personnel avant d'accéder à l'application.

Messages d'erreur

"Ce lien d'activation est invalide ou a expiré".
Cause : le lien a été utilisé une fois (il est à usage unique) ou 72 heures se sont écoulées. Solution : allez sur la fiche de l'employé et utilisez "Renvoyer l'invitation" pour générer un nouveau lien.

Conseils

  • Le lien d'activation expire après 72 heures. Si l'employé ne l'a pas utilisé, allez sur sa fiche et appuyez sur "Renvoyer l'invitation".
  • Le mot de passe temporaire n'est visible qu'une seule fois. Il est impossible de le retrouver ensuite. Si l'employé l'a perdu, désactivez le compte et recréez-en un.
  • Un employé sans email peut toujours bénéficier d'une invitation si vous mettez son email à jour plus tard dans sa fiche.
